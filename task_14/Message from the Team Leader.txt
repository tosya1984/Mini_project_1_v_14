Повідомлення від Тім Ліда (Ольга Сидорчук)  
Дата: 24 вересня 2024 року

Огляд проекту:  
Ми розробляємо платформу для продажу товарів ручної роботи, де користувачі можуть переглядати товари, створювати облікові записи, робити замовлення і продавати свої вироби. Додаток повинен мати надійний бекенд для управління замовленнями, користувачами та товарами, статичний фронтенд для перегляду товарів і API для інтеграції з платіжними системами.

Задача:  
Необхідно задеплоїти всі сервіси: Redis для кешування сесій, RDS для бази даних, а також статичний фронтенд через S3 і CloudFront згідно з архітектурою. Ваше завдання — забезпечити оптимальну продуктивність та відповідність бюджету.

База даних:  
PostgreSQL буде використовуватися для зберігання даних про користувачів, товари та замовлення. Redis буде використовуватися для кешування сесій користувачів і результатів пошуку товарів для підвищення швидкості.

Докеризація:  
Усі сервіси повинні бути докеризовані для забезпечення легкої масштабованості та зручності розгортання.

CI/CD процес:  
Налаштуйте окремі CI/CD пайплайни для автоматизації деплою бекенду і фронтенду.

Важливі нотатки:  
- Фронтенд повинен бути статичним і зберігатися на S3 з кешуванням через CloudFront для прискорення завантаження сторінок.
- Використовуйте архітектуру на основі EC2 або ECS, як зазначено на діаграмі.
- Забезпечте дотримання бюджету та продуктивності відповідно до вимог клієнта.
